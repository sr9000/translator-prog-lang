translator формат хранения объектного кода.
весь код разбиваетс на контексты.
T_object_code: env1; env2; env3...
контекст состоит из двух частей никак не отделённых друг от друга
code и data. Таким образом имеем:
T_object_code: {code data}1; {code data}2; {code data}3...
data - это набор ячеек для всех переменных, используемых  контексте
T_object_code: {code {el1 el2 el3 ...}}1; {code {el1 el2 el3 ...}}2; {code {el1 el2 el3 ...}}3...
каждый элемент универсально описывает любую переменную
мы имеем:
1 - бор
2 - множество(тот же бор)
3 - объект
4 - фрейм
5 - число
6 - строка
7 - файл
8 - вектор
все данные хранятся в оболочке Var
данная оболочка обладает одним полем:
указатель на оболочку-счётчик ссылок
и это оболочка уже имеет
char тип хранимого значения
int счётчик ссылок на данный объект
void* указатель на хранимую переменную
type:
b бор
a множество(тот же бор)
o объект
f фрейм
i число
s строка
c файл, code(код исходный и транслированный)
v вектор
